# OctoAcme Project Management Docs

Welcome to the OctoAcme project management documentation. This README summarizes how we run projects at OctoAcme and provides quick links to detailed process documents in the `docs/` folder.

## Project Management Process Summary

OctoAcme uses an iterative, customer-first approach supported by clear roles, lightweight artifacts, and regular communication. Projects follow a structured lifecycle guided by the following core principles: prioritize customer value, deliver in small testable increments, maintain clear ownership, make data-informed decisions, and foster psychological safety.

Core processes include:

- **Initiation:** Define goals, measure success, align stakeholders, and create a project one-pager.
- **Planning:** Break work into shippable increments, prioritize the backlog, and plan releases.
- **Execution & Tracking:** Use project boards, regular standups, and reviews to ensure progress and surface blockers.
- **Risk Management & Communication:** Maintain a simple risk register and use clear communication rhythms to escalate or resolve issues.
- **Release & Deployment:** Standardize releases with consistent checklists and pre-release criteria.
- **Retrospective & Continuous Improvement:** After each milestone, capture learnings and improvements, then feed them back into processes.
